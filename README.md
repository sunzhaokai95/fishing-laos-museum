# 钓鱼佬博物馆

一座沿唯一参观路线展开的中文线上钓鱼博物馆。项目以中国钓鱼史为起点，依次呈现鱼类、钓具、技法、人物、鱼文化以及钓获之后的法规与环境问题。

## 当前内容

- 31 个中国钓鱼史时间轴节点
- 806 种鱼、794 张鱼类图片、215 个科
- 钓竿、轮、线、钩、漂坠、饵和辅助工具资料
- 21 项饵技、竿技、漂技和鱼技
- 6 类钓者身份
- 诗画典故、钓鱼语言和玄学标本
- 5 个国家和地区的管理模型及 5 类钓获后判断

参观路线固定为：

`首页 -> 序厅 -> 历史 -> 鱼类 -> 钓具 -> 技法 -> 人物 -> 文化 -> 规则与环境 -> 尾厅`

## 技术栈

- React 19
- React Router
- Vite
- Lucide React
- 普通 CSS

## 本地运行

```bash
pnpm install
pnpm dev
```

## 验证

```bash
pnpm lint
pnpm test -- --run
pnpm build
```

构建前会运行 `pnpm sync:content`，将策展内容同步到 `public/content`。

## 主要目录

- `src/halls/`：序厅、七个展厅和尾厅组件
- `src/components/`：导航、路线进度、抽屉和翻页组件
- `src/editorial.css`：当前编辑式博物馆视觉层
- `public/content/data/`：鱼类、百科、时间轴和馆藏数据
- `public/content/assets/`：网站使用的本地图片
- `DESIGN.md`：视觉系统和各厅设计语言
- `PRODUCT.md`：产品定位与不可破坏的体验原则
- `design/current/`：当前桌面端和手机端页面截图
- `GEMINI_DESIGN_BRIEF.md`：供 Gemini 从 GitHub 读取的线上视觉优化任务书

## 设计与内容边界

- 不增加多条路线、参观时长、积分、答题或解锁机制。
- 不把真实数据替换成组件内的模拟数据。
- 前台不显示内部“研究与来源”工作流。
- 必须保留搜索、筛选、渐进加载、详情抽屉、上一厅／下一厅和旧链接兼容。
- 兼容 1440×900 桌面端和 390×844 手机端，并支持键盘与降低动态模式。
